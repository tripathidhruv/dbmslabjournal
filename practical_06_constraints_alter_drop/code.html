Group B | Practical 06

Constraints &
Schema Evolution

Mastering data integrity through Primary and Foreign Keys, and the art of dynamic table structural modifications using ALTER and DROP commands.

key Data Integrity

Constraints are rules enforced on data columns of a table. These are used to limit the type of data that can go into a table. This ensures the accuracy and reliability of the data in the database.

PRIMARY KEY Unique identifier for each record. Cannot be NULL.
FOREIGN KEY Points to a Primary Key in another table, creating a link.
mysql terminal - schema_definition.sql content_copy
-- Creating tables with constraints
CREATE TABLE Departments (
  dept_id INT PRIMARY KEY,
  dept_name VARCHAR(50) NOT NULL
);

CREATE TABLE Employees (
  emp_id INT PRIMARY KEY,
  emp_name VARCHAR(100),
  dept_id INT,
  FOREIGN KEY (dept_id) REFERENCES Departments(dept_id)
);

edit_square ALTER & DROP

ADD COLUMN

Expand existing tables without losing current data.

ALTER TABLE Employees ADD salary INT;

MODIFY COLUMN

Adjust data types or constraint attributes.

ALTER TABLE Employees MODIFY emp_name VARCHAR(150);

DROP TABLE

Permanent removal of structure and data.

DROP TABLE Temp_Records;
warning

DROP is irreversible. Always verify references before execution.
