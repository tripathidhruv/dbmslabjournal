Practical 12: Cursors

A cursor is a pointer to the context area. PL/SQL controls the context area through a cursor. It holds the rows returned by a SQL statement.

auto_fix_high

Implicit Cursors

Automatically created by Oracle whenever an SQL statement is executed, when there is no explicit cursor for the statement.

edit_note

Explicit Cursors

Defined by programmers for queries that return more than one row. These offer more control over data processing and require a specific lifecycle.

data_object

The Cursor Lifecycle

01

DECLARE

Defines the cursor name and the associated SQL query.

02

OPEN

Allocates memory and executes the SQL query.

03

FETCH

Retrieves data from the cursor into local variables.

04

CLOSE

Releases the allocated memory area.

Practical Implementation

cursor_loop.sql 
-- Cursor with Loop Implementation
DECLARE
    c_id students.id%TYPE;
    c_name students.name%TYPE;
    CURSOR c_students IS 
        SELECT id, name FROM students;
BEGIN
    OPEN c_students;
    LOOP
        FETCH c_students INTO c_id, c_name;
        EXIT WHEN c_students%NOTFOUND;
        
        dbms_output.put_line(c_id || ' ' || c_name);
    END LOOP;
    CLOSE c_students;
END;
