Practical 10

Write and execute PL/SQL stored procedure and function to manage complex database operations and return computed values.

CO-MAPPED DBMS_C3

Procedures vs Functions

In DBMS, Stored Procedures are groups of statements that can be saved and reused. They are typically used for administrative tasks or complex business logic that doesn't necessarily return a single value.

Conversely, Functions are specialized procedures that MUST return a value. They are often used in calculations and can be called directly within SQL queries (e.g., SELECT statements).

The Delimiter

In MySQL, the default delimiter is ;. When writing PL/SQL, we must temporarily change it to something like // to allow the entire block to be compiled as a single unit. 

DELIMITER //

CREATE PROCEDURE UpdateSalary(
    IN emp_id INT,
    IN increment DECIMAL(10,2)
)
BEGIN
    UPDATE Employees 
    SET salary = salary + increment
    WHERE id = emp_id;
    
    -- Log the transaction
    INSERT INTO AuditLog(emp_id, log_time) 
    VALUES (emp_id, NOW());
END //

DELIMITER ;
terminal Query Output
mysql> CALL UpdateSalary(101, 500.00);
Query OK, 1 row affected (0.01 sec)
mysql> SELECT * FROM Employees WHERE id = 101;
