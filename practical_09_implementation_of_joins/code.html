Implementation of Joins

Inner Join

Returns records that have matching values in both tables. This is the most common type of join.

SELECT S.NAME, D.DEPT_NAME
FROM Students S
INNER JOIN Departments D
ON S.DEPT_ID = D.DEPT_ID;

Execution Result

NAME DEPT_NAME
Alan TuringComputer Science
Grace HopperMathematics
Ada LovelaceComputer Science
02.

Left Join

Returns all records from the left table, and the matched records from the right table. Records with no match in the right table show NULL.

SELECT S.NAME, D.DEPT_NAME
FROM Students S
LEFT JOIN Departments D
ON S.DEPT_ID = D.DEPT_ID;

Execution Result

NAME DEPT_NAME
Alan TuringComputer Science
Grace HopperMathematics
Ada LovelaceComputer Science
John von NeumannNULL
03.

Right Join

Returns all records from the right table, and the matched records from the left table. Departments with no students will result in NULL.

SELECT S.NAME, D.DEPT_NAME
FROM Students S
RIGHT JOIN Departments D
ON S.DEPT_ID = D.DEPT_ID;

Execution Result

NAME DEPT_NAME
Alan TuringComputer Science
Ada LovelaceComputer Science
Grace HopperMathematics
NULLPhysics
04.

Natural Join

A Natural Join creates an implicit join clause for you based on the common columns in the two tables being joined. In this case, both tables share DEPT_ID.

SELECT *
FROM Students
NATURAL JOIN Departments;
