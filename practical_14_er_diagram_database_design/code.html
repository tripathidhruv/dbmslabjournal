Group D — Project Construction

Practical 14: ER Diagram & Custom DB Design

Executing the final normalized creation architecture of the target custom project proposal.

Aim

Objective

The aim of this practical is to synthesize the previously established structural SRS into a fully operational Database Design. The student will draft an Entity Relational Diagram and commit the final schema design confirming it meets BCNF normalization parameters and fully connects the overarching application dependencies.

Theory & Background

Core Concepts

Final ER Diagram Construction

Every strong entity defined in the SRS becomes a relational table construct. Multivalued traits become linking structures. ER acts as the final layout agreement linking the raw specifications to the final Codd-normalized environment deployment.

Design Verification

The resulting final schema must be evaluated against all DML use cases envisioned. If 'Finding a patient history' takes 4 excessive sub-joins, the schema must be evaluated on balance vs speed vs normalization.

Procedure / Algorithm

Step-by-Step
  1. 01.Map entities strictly against SRS documents.
  2. 02.Assign and verify Primary and Partial keys.
  3. 03.Enforce strict Relational lines avoiding cross-dependency loops.
  4. 04.Execute the final unifying schema codebase.
  5. 05.Export and construct final Diagram.

Viva Voce

Q & A
Q1.How does an ER diagram support Project Management?
Ans: It acts as the visual contract between software developers creating the frontend interface, and database architects maintaining the backend structures.
Q2.What does an entity correspond to in physical SQL layout?
Ans: An entity is translated into a structured TABLE.
Q3.What indicates a 1:N relationship visually in an ER diagram?
Ans: Commonly a 'Crow's Foot' or simply 1 and N labels extending between a Diamond relationship box toward the rectangle entity boundaries.
Q4.Can an ER Diagram have errors?
Ans: Yes. Most commonly 'Chasm traps' or 'Fan traps' where relationships are ambiguous, meaning the eventual SQL schema will fail to associate data correctly across multiple joined links.
